## AC-001 — Create a product with SKUs

### TS-002-001 · Happy path — product with single SKU created

**Preconditions:**
- The inventory-service is running and the database is available.
- A valid `X-Tenant-ID` header is provided.

**Action:**
`POST /products` with body:
```json
{
  "name": "Widget Pro",
  "description": "A high-quality widget.",
  "skus": [
    { "label": "Blue / M", "price_minor": 1999, "initial_stock": 50 }
  ]
}
```

**Expected outcome:**
- HTTP 201 response.
- Response body contains a top-level `id` (UUID).
- Response `skus` array has exactly one item with an assigned `id`, `label: "Blue / M"`, `price_minor: 1999`.
- `GET /stock/{sku.id}` returns `total: 50, reserved: 0, available: 50`.

---

### TS-002-002 · Happy path — product with multiple SKUs created

**Preconditions:**
- Valid tenant context.

**Action:**
`POST /products` with `skus` containing three items with different labels, prices, and initial stock values.

**Expected outcome:**
- HTTP 201.
- Response `skus` array has exactly three items, each with a unique `id`.
- Each SKU's stock is independently queryable via `GET /stock/{skuId}`.

---

### TS-002-003 · Validation failure — missing product name

**Preconditions:**
- Valid tenant context.

**Action:**
`POST /products` with body `{ "skus": [{ "label": "A", "price_minor": 100, "initial_stock": 10 }] }` (no `name` field).

**Expected outcome:**
- HTTP 422 response.
- Response body contains a `code` field and a descriptive `message`.

---

### TS-002-004 · Validation failure — product with no SKUs

**Preconditions:**
- Valid tenant context.

**Action:**
`POST /products` with body `{ "name": "Empty Product", "skus": [] }`.

**Expected outcome:**
- HTTP 422 response (at least one SKU is required).

---

### TS-002-005 · Validation failure — SKU with negative price

**Preconditions:**
- Valid tenant context.

**Action:**
`POST /products` with a SKU where `price_minor: -1`.

**Expected outcome:**
- HTTP 422 response.

## AC-002 — List products with pagination

### TS-002-006 · Happy path — default pagination

**Preconditions:**
- 25 active products exist for the tenant.

**Action:**
`GET /products` with only the `X-Tenant-ID` header.

**Expected outcome:**
- HTTP 200.
- `data` array contains 20 items (default `per_page`).
- `meta.total` is 25, `meta.page` is 1, `meta.per_page` is 20.

---

### TS-002-007 · Pagination — second page

**Preconditions:**
- 25 active products exist for the tenant.

**Action:**
`GET /products?page=2&per_page=20`.

**Expected outcome:**
- HTTP 200.
- `data` array contains 5 items.
- `meta.page` is 2.

---

### TS-002-008 · Filter — in_stock_only=true excludes zero-stock products

**Preconditions:**
- 3 products exist: Product A (available: 10), Product B (available: 0), Product C (available: 5).

**Action:**
`GET /products?in_stock_only=true`.

**Expected outcome:**
- HTTP 200.
- `data` contains Product A and Product C only. Product B is absent.
- `meta.total` reflects the filtered count (2, not 3).

---

### TS-002-009 · Empty catalogue returns empty list

**Preconditions:**
- No products exist for the tenant.

**Action:**
`GET /products`.

**Expected outcome:**
- HTTP 200.
- `data` is an empty array `[]`.
- `meta.total` is 0.

---

### TS-002-010 · Tenant isolation — products from another tenant are not returned

**Preconditions:**
- Tenant A has 3 products. Tenant B has 2 products.

**Action:**
`GET /products` with `X-Tenant-ID: <Tenant B UUID>`.

**Expected outcome:**
- HTTP 200.
- `data` contains only the 2 products belonging to Tenant B.

---

## AC-003 — Get a single product by ID

### TS-002-011 · Happy path — product found

**Preconditions:**
- A product with ID `prod-uuid-1` exists for the tenant with 2 SKUs.

**Action:**
`GET /products/prod-uuid-1`.

**Expected outcome:**
- HTTP 200.
- Response includes `id`, `name`, `skus` array (2 items), and `created_at`.
- Each SKU in the response includes `id`, `label`, `price_minor`, `stock_level`.

**Preconditions:**

**Action:**
`GET /products/00000000-0000-0000-0000-000000000099`.

**Expected outcome:**
- HTTP 404 response.
- Response body contains `code` and `message` fields.

---

### TS-002-013 · Tenant isolation — cannot retrieve another tenant's product

**Preconditions:**
- `prod-uuid-1` belongs to Tenant A.

**Action:**
`GET /products/prod-uuid-1` with `X-Tenant-ID: <Tenant B UUID>`.

**Expected outcome:**
- HTTP 404 (product not visible to Tenant B).

## AC-004 — Query stock level for a SKU

### TS-002-015 · SKU not found

**Action:**
`GET /stock/00000000-0000-0000-0000-000000000099`.

**Expected outcome:**
- HTTP 404.

---

### TS-002-016 · Stock level reflects a reservation

**Preconditions:**

## AC-005 — Reserve stock (internal)

### TS-002-017 · Happy path — reservation created

**Preconditions:**
- SKU-A: total: 10, reserved: 0 (available: 10).
- SKU-B: total: 5, reserved: 0 (available: 5).

**Action:**
`POST /stock/reserve` with:
```json
{
  "order_id": "order-uuid-1",
  "lines": [
    { "sku_id": "SKU-A", "quantity": 3 },
    { "sku_id": "SKU-B", "quantity": 2 }
  ]
}
```

**Expected outcome:**
- HTTP 200.
- Response contains `reservation_id` (UUID) and `expires_at` (ISO 8601 timestamp, ~15 min from now).
- `GET /stock/SKU-A` shows `reserved: 3, available: 7`.
- `GET /stock/SKU-B` shows `reserved: 2, available: 3`.

---

### TS-002-018 · Partial stock — one SKU insufficient

**Preconditions:**
- SKU-A: available: 5. SKU-B: available: 1.

**Action:**
`POST /stock/reserve` requesting SKU-A qty 3 and SKU-B qty 2.

**Expected outcome:**
- HTTP 409.
- Response: `code: "STOCK_CONFLICT"`, `conflicts` array contains exactly one entry for SKU-B with `requested: 2, available: 1`.
- No reservation record is created. Stock levels for SKU-A are unchanged.

---

### TS-002-019 · All SKUs out of stock

**Preconditions:**
- SKU-A: available: 0. SKU-B: available: 0.

**Action:**
`POST /stock/reserve` requesting any quantity of SKU-A and SKU-B.

**Expected outcome:**
- HTTP 409.
- `conflicts` array contains both SKU-A and SKU-B.

---

### TS-002-020 · Concurrent reservations — no overselling

**Preconditions:**
- SKU-X: total: 5, reserved: 0 (available: 5).

**Action:**
Two concurrent `POST /stock/reserve` requests, each requesting quantity 4 for SKU-X, sent simultaneously.

**Expected outcome:**
- Exactly one request returns HTTP 200 (reservation created).
- The other returns HTTP 409 with `STOCK_CONFLICT` for SKU-X.
- After both complete: `GET /stock/SKU-X` shows `reserved: 4, available: 1` (not `reserved: 8`).

---

## AC-006 — Deduct reserved stock on order confirmation

### TS-002-021 · Happy path — reservation converted to deduction

**Preconditions:**
- An active reservation `res-uuid-1` exists for SKU-A qty 3 (total: 10, reserved: 3, available: 7).

**Action:**
`POST /stock/deduct` with `{ "reservation_id": "res-uuid-1" }`.

**Expected outcome:**
- HTTP 200.
- `GET /stock/SKU-A` returns `total: 7, reserved: 0, available: 7` (both `total` and `reserved` decremented by 3).

---

### TS-002-022 · Idempotency — double deduction rejected

**Preconditions:**
- Reservation `res-uuid-1` has already been deducted (status: `converted`).

**Action:**
A second `POST /stock/deduct` with `{ "reservation_id": "res-uuid-1" }`.

**Expected outcome:**
- HTTP 409.
- Stock levels are not changed a second time.

---

### TS-002-023 · Reservation not found

**Preconditions:**
- No reservation with the given ID exists.

**Action:**
`POST /stock/deduct` with a non-existent `reservation_id`.

**Expected outcome:**
- HTTP 409 (reservation not found is treated as a conflict to prevent silent data loss).

---

## AC-007 — Release a reservation on saga compensation

### TS-002-024 · Happy path — reservation released

**Preconditions:**
- An active reservation `res-uuid-2` exists for SKU-A qty 5 (total: 20, reserved: 5, available: 15).

**Action:**
`POST /stock/reservations/res-uuid-2/release`.

**Expected outcome:**
- HTTP 204 (no body).
- `GET /stock/SKU-A` returns `reserved: 0, available: 20`.

---

### TS-002-025 · Release non-existent reservation — treated as no-op

**Preconditions:**
- No reservation exists with the given ID.

**Action:**
`POST /stock/reservations/00000000-0000-0000-0000-000000000099/release`.

**Expected outcome:**
- HTTP 404.
- No change to any stock level (caller treats 404 as a no-op success in the saga compensation).

## AC-008 — Service health and observability

### TS-002-026 · /health returns 200 when service is running

**Preconditions:**
- The service process is running (database may or may not be available).

**Action:**
`GET /health`.

**Expected outcome:**
- HTTP 200.
- Response body: `{ "status": "ok" }`.

---

### TS-002-027 · /ready returns 200 when database is reachable

**Preconditions:**
- Service is running. Database is reachable.

**Action:**
`GET /ready`.

**Expected outcome:**
- HTTP 200.
- Response body contains `{ "status": "ready" }`.

---

### TS-002-028 · /ready returns 503 when database is unreachable

**Preconditions:**
- Service is running. Database connection is severed (e.g. postgres container stopped).

**Action:**
`GET /ready`.

**Expected outcome:**
- HTTP 503.
- Response body indicates the database is unavailable.

---

### TS-002-029 · /metrics returns Prometheus format

**Preconditions:**
- Service has received at least one HTTP request.

**Action:**
`GET /metrics`.

**Expected outcome:**
- HTTP 200.
- `Content-Type` header contains `text/plain`.
- Response body contains standard Prometheus metric lines (e.g. `http_requests_total`).
